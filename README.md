# Система классификации обращений в техническую поддержку

## Обзор проекта
Система автоматической классификации обращений в техническую поддержку с веб-интерфейсом для управления тикетами.

> **Цель проекта**: Автоматизировать процесс обработки обращений в техподдержку, сократить время ответа и улучшить качество обслуживания.

## Функциональность

### Классификация обращений

**Срочный** - критические проблемы, требующие немедленного решения

*Не срочный* - положительные отзывы, благодарности

**Средняя важность** - нейтральные обращения

*Важный* - вопросы, требующие ответа

**Можно отложить** - предложения по улучшению

***СПАМ/Грубый*** - токсичные сообщения и спам

### Веб-интерфейс (Streamlit)
*Отправка обращений* - форма для создания новых тикетов

*Управление тикетами* - просмотр, фильтрация, ответы

*Автоматическая классификация* -  модель определяет категорию

*Статусы* - Новый / В работе / Закрыт

*Фильтрация* - по статусу и категории

### API сервис (FastAPI)
`/predict` - эндпоинт для классификации текста

Возвращает категорию и уверенность предсказания

Интеграция с веб-интерфейсом

## Модель классификации

### Предобработка текста
1. Приведение к нижнему регистру
2. Удаление спецсимволов и цифр
3. Удаление стоп-слов
4. Лемматизация (pymorphy2)
5. TF-IDF векторизация

### Архитектура модели
*Алгоритм*: Random Forest (100 деревьев)

*Векторизация*: TF-IDF (5000 фич, биграммы)

*Точность*: ~85-90% (зависит от датасета)

### Датасет
<sub>18,000 примеров (сбалансированных)</sub>

<sup>Источники: отзывы, токсичные комментарии, вопросы</sup>

***6 категорий с равномерным распределением***
